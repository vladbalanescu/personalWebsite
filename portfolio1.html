  • Java Picture
    My very first university assignment consisted of a .txt file which contained 40,000 digits in range 0-9. I had to assign a specific color to each digit and plot them in a Java graphical window, so that the frame will display a picture of 200x200 pixels. I was provided with the input file and given the task of drawing the original picture and a secondary one, which had to render only the boundaries of the initial one, I came up with these 2 pictures on the left.   • Technologies used: Ruby, Gems(Colored, Colorize)

  • Battleship Game
    A Battleship game was the first task which required me to write code in Ruby. I stored all the information requested by the game in two 2D arrays (one for initial state of the game, and one for the active state of the game), where ships positions were marked by 1 and the water ones by 0. Using more gems, the display consisted of coloured text, where the board was colored in red if a shot hit a ship or blue if it was a miss. Different further futures were implemented, such as signaling when a ship was sunk or displaying the score at the end of the game.

  • Global Engineering Challenge
    A one week real life engineering experience where together with a team of six engineers, I had to provide a solution to an environmental issue (Water Pumping in Gorka district, Nepal). Our task consisted in helping the villagers in finding a solution to bring pure drinkable water to a settlement on the top of a mountain. Hence, our project idea was based on a Ram Pump, which would definetly solve this issue, requiring no power supply. Hover the image to see the award for this project!

  • Chess Game
    On this project I had to implement a simple version of a Chess game, where a player loses if his king is taken by the opponent. It was a really challenging and complex project, where the task was to implement a 1-player mode and a 2-players one. If 1-player mode was chosen, the player had to select the difficulty of the game. I implemented "check" checker for the king, which displays a 'Try again!' message if the next selected move threatens player's-to-move king. Click on this image to watch a full demo!

  • Rambler Searcher
    In my Artificial Intelligence module, I got the task: 'work out the best walking route from a start point to a goal point, given a terrain map of the area in the form of a .pgm file'. The input was provided as a 2D array, where each value represented a certain height. The Rambler had to choose the easiest way to achieve its goal (climbing as least as it can, while driving in the field). Hover the image to see the content of the input file, opened in IrfanView.

  • Technologies used: Java

  • Intelligent Warrior
    My second Artificial Intelligence project consisted of a simple game where I had to prove my understanding about game planning with STRIPS. The game template involved a warrior and 3 tools: a ladder, a hook and a rope located at Start. If there was no snake in the pit, the warrior had to plan to take the ladder, lower it into the pit and take the treasure, otherwise, carry the rope and the hook at pit entrance, lower the rope into the pit and hook treasure without being in danger. Hover the image for the result or check it in gallery!
